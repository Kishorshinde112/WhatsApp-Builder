Build a production-oriented MVP web app for WhatsApp campaign operations.

Project goal:
Create a clean, reliable, operator-grade web app that is better than Deropo in execution quality, while focusing ONLY on 3 core modules:
1. Campaigns
2. Contacts
3. Tracking

This is not a generic marketing tool. It is a focused WhatsApp campaign operations dashboard for sending personalized campaigns and tracking outcomes with clarity.

Core product philosophy:
- Be better than Deropo by being cleaner, safer, and easier to trust
- Do NOT add chatbot, CRM, AI reply, warmup, group grabber, autoresponder, or multi-account rotation in v1
- Win through reliability, tracking clarity, data quality, and UX
- The app should feel like a serious internal operations product, not a flashy template
- The app should be ready to later connect with Green API or Evolution API

Primary user story:
A user uploads/imports contacts, creates a campaign using a template like:
"Hi {{name}}, ..."
sets a delay between messages, previews sample outputs, launches the campaign, and later monitors live status:
sent / delivered / read / failed / noAccount
with logs, filters, and export.

Tech stack requirements:
- Full-stack web app
- Use Node.js + Express backend
- Use SQLite for MVP database
- Use simple server-rendered UI with EJS OR a clean React frontend with Express API
- Prefer the simplest stack that Replit can generate fast and that I can later move to GitHub and deploy on Oracle VPS
- Add a `.env.example`
- Add a clean `README.md`
- Add scripts for local run
- Structure code cleanly for future VPS deployment with PM2 or Docker
- Keep API integration abstracted behind a provider service layer
- Add a mock provider initially if real provider credentials are not configured

Provider abstraction:
Design the backend so that WhatsApp provider can be swapped later.
Create provider interface/service like:
- sendMessage(contact, renderedMessage)
- getStatus(messageId)
- handleWebhook(payload)

Prepare for Green API / Evolution API integration later.
